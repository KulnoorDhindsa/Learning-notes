# Programming in C++
- **Classes**: They are collection of categories shared by its objects; its a *blue-print* for its objects.
- **Objects**: Objects are entries or *instances* in a class.
- **Encapsulation**: Data enclosed in a *class* which is unique to that class **only**.
>When data is hidden from programmer (dues to encapsulation), its called **information hiding**.
- **Abstraction**: Hides complexity of *how does this work* and displays only essential features of object to the user.
- **Inheritance**: This is like *hierarchical classification*, a class acquires properties of its parents class, with additions of its own.

- **Polymorphism**: 
    - **Operator Overloading**: When an operator perform different tasks. E.g: `sum` with integers adds integers, and with strings, `sum` concatenates to form new string
    - **Function Overloading**: When a function performs different tasks. e.g: `draw()` function draws a circle when asled a circle and a traingle when asked a triangle.
- **Binding**: Linking code with a called procedure that is to be executed.
    - **Dynamic Binding**: Code isn't known untill its called.
